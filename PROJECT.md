# وثائق المشروع — maghzaccount-pro

> **الغرض:** يقدم هذا الملف النظرة العامة الشاملة على المشروع، أهدافه، جمهوره المستهدف، ميزاته، وخارطة طريق التطوير التفصيلية. هو المدخل الأول لأي شخص يريد فهم "ما هذا المنتج ولماذا يوجد".

---

## 1. نظرة عامة (Overview)

**maghzaccount-pro** هو نظام ERP (Enterprise Resource Planning) محاسبي متكامل احترافي موجه للمنشآت الصغيرة والمتوسطة في العالم العربي. يُبنى بتقنيات حديثة وتصميم مريح بصرياً، ويدعم اللغة العربية (RTL) كلغة أساسية مع الإنجليزية، ويقدم وظائف محاسبية متوافقة مع المعايير الدولية (IFRS).

يقدم النظام حلاً مالياً وإدارياً متكاملاً:
- **المعايير المحاسبية المزدوجة** (Double-Entry Bookkeeping).
- **نظام ضريبة القيمة المضافة (VAT)** مرن وقابل للتخصيص.
- **دعم متعدد العملات** مع الريال اليمني (YER) كعملة افتراضية.
- **شجرة حسابات جاهزة** متوافقة مع IFRS.
- **تقارير متقدمة وذكية وتحليلية** لكل وحدة على حدة.
- **لوحة تحكم رئيسية (Dashboard)** تعرض KPIs من كل الوحدات.
- **تصميم عربي/إنجليزي** مع خطوط احترافية (Cairo + Inter).
- **وضع فاتح وداكن** (Light / Dark Mode).

## 2. الرؤية والرسالة

### الرؤية (Vision)
> "تمكين كل منشأة عربية — مهما كان حجمها — من إدارة مواردها المالية والتشغيلية باحترافية، باستخدام أدوات رقمية عصرية وسهلة الاستخدام، مع تقارير متقدمة تساعد على اتخاذ القرارات."

### الرسالة (Mission)
- تبسيط العمليات المحاسبية المعقدة.
- توفير نظام ERP كامل يعمل offline (Electron) و online (Web/SaaS مستقبلي).
- تقديم تقارير مالية وتحليلات ذكية تفوق الحلول الأجنبية.
- دعم كامل للبيئة العربية (RTL، التواريخ الهجرية، العملات المحلية).

## 4. الأقسام والوحدات المنفصلة (11 Module Architecture)

يتبنى maghzaccount-pro **معمارية مقسمة (Modular Architecture)** حيث كل وحدة مستقلة تماماً:

```
┌─────────────────────────────────────────────┐
│           maghzaccount-pro                    │
├─────────────────────────────────────────────┤
│  01. الأساس (Core)                            │
│     → تهيئة النظام، الشركات، العملات         │
├─────────────────────────────────────────────┤
│  02. الإعدادات (Settings)                     │
│     → إعدادات النظام، الضريبة، الفروع        │
├─────────────────────────────────────────────┤
│  03. المستخدمين والصلاحيات (Auth & RBAC)       │
│     → تسجيل دخول، أدوار، صلاحيات              │
├─────────────────────────────────────────────┤
│  04. الحسابات (Accounting / GL)               │
│     → شجرة حسابات، قيود يومية، تقارير مالية   │
├─────────────────────────────────────────────┤
│  05. المخازن (Inventory)                       │
│     → منتجات، مستودعات، جرد، تحويلات          │
├─────────────────────────────────────────────┤
│  06. المبيعات (Sales)                          │
│     → فواتير، VAT، عروض أسعار، عملاء           │
├─────────────────────────────────────────────┤
│  07. المشتريات (Purchases)                     │
│     → فواتير، أوامر شراء، موردين               │
├─────────────────────────────────────────────┤
│  08. التصنيع (Manufacturing)                   │
│     → BOM، أوامر تشغيل، تكاليف                 │
├─────────────────────────────────────────────┤
│  09. الموظفين (HR & Payroll)                   │
│     → حضور، رواتب، نهاية خدمة                  │
├─────────────────────────────────────────────┤
│  10. علاقات العملاء (CRM)                      │
│     → فرص، مهام، مكالمات، تحليلات              │
├─────────────────────────────────────────────┤
│  11. التقارير (Reports & Analytics)            │
│     → Dashboard رئيسي، تقارير متقدمة، ذكاء    │
└─────────────────────────────────────────────┘
```

**كل وحدة تحتوي على:**
- قاعدة بيانات مستقلة (Schema)
- API مستقل (Database Adapters)
- مكونات React مستقلة
- **تقارير متقدمة خاصة بالوحدة**
- ترجمة مستقلة (i18n namespace)

## 5. نظام التقارير المتقدم (Advanced Reporting System)

### 5.1 التقارير لكل وحدة

| الوحدة | التقارير المتاحة | النوع |
|--------|-----------------|-------|
| **الحسابات** | ميزان المراجعة، الميزانية العمومية، قائمة الدخل، تدفق نقدي، دفتر أستاذ، مقارنة فترات، نسب ربحية | قياسي + تحليلي |
| **المخازن** | بطاقة صنف، جرد مخزون، تحويلات، الحد الأدنى، بطء الحركة، تكلفة المخزون | عملياتي + ذكي |
| **المبيعات** | فواتير، عمر الديون (A/R)، أداء العملاء، المنتجات الأكثر مبيعاً، تحليل المواسم، VAT | تفصيلي + تحليلي |
| **المشتريات** | فواتير، أوامر شراء، عمر الديون (A/P)، مقارنة أسعار، VAT | تفصيلي + تحليلي |
| **التصنيع** | تكلفة إنتاج، أوامر تشغيل، استهلاك مواد، variance analysis، كفاءة خط | تحليلي + ذكي |
| **الموظفين** | مسير رواتب، حضور، إجازات، نهاية خدمة، تحليل الرواتب | قياسي + تحليلي |
| **CRM** | قمع مبيعات، أداء المندوبين، conversion rate، سجل نشاطات | تحليلي + ذكي |

### 5.2 التقارير المركزية (Central Reports)

- **لوحة التحكم الرئيسية (Main Dashboard):** KPIs من كل الوحدات في صفحة واحدة.
- **التقرير المالي المتكامل:** P&L + Balance Sheet + Cash Flow معاً.
- **التنبيهات الذكية:** فواتير مستحقة، مخزون منخفض، رواتب مستحقة.
- **التوقعات (Forecasting):** توقع المبيعات والأرباح بناءً على المتوسطات.
- **المقارنات التاريخية:** مقارنة سنة مع سنة، ربع مع ربع.

### 5.3 تصدير التقارير

| الصيغة | المكتبة | الاستخدام |
|--------|---------|----------|
| **PDF** | `jspdf` + `jspdf-autotable` (lazy-loaded) | فواتير، تقارير رسمية |
| **Excel** | `xlsx` (SheetJS) | تحليلات، بيانات خام |
| **CSV** | Native | استيراد/تصدير سريع |

## 12. خارطة الطريق الزمنية التفصيلية (Roadmap Timeline)

### المرحلة 0: الأساس (Foundation) — أسابيع 1-2
- تهيئة المشروع (React 19 + TypeScript + Vite + Electron + Tailwind).
- نظام التصميم (Cairo/Inter + Light/Dark + CSS Variables).
- نظام الترجمة (i18n ar/en + RTL).
- طبقات البيانات (PostgreSQL + Drizzle + Mock).
- إدارة الشركات والعملات المتعددة.

### المرحلة 1: الإعدادات + المستخدمين — أسبوع 3
- إعدادات النظام العامة (VAT، العملات، الفروع).
- نظام المستخدمين والأدوار (RBAC: Super Admin, Accountant, Manager, Viewer).
- تسجيل الدخول والأمان.

### المرحلة 2: الحسابات (Accounting) — أسابيع 4-6
- شجرة حسابات جاهزة (IFRS-compliant: Assets, Liabilities, Equity, Revenue, Expenses).
- قيود اليومية المزدوجة مع validation تلقائي (debits = credits).
- **تقارير:** Trial Balance, Balance Sheet, P&L, Cash Flow, Account Ledger.
- **تحليلات:** Comparative Analysis, Profitability Ratios.

### المرحلة 3: المخازن (Inventory) — أسابيع 7-8
- منتجات، تصنيفات، مستودعات، وحدات قياس.
- تحويلات مخزنية، الجرد والتسوية.
- **تقارير:** Item Ledger, Stock Count, Transfers, Low Stock Alert, Slow Moving, Inventory Valuation.

### المرحلة 4: المبيعات (Sales) — أسابيع 9-11
- فواتير مبيعات + VAT + عروض أسعار (Quotations).
- مرتجعات مبيعات.
- حسابات العملاء (A/R Aging).
- التحصيل والمقبوضات.
- **تقارير:** Sales Invoices, Customer Statement, Top Customers, Seasonal Analysis, A/R Aging, VAT Report.
- **تحليلات:** Sales Funnel, Rep Performance.

### المرحلة 5: المشتريات (Purchases) — أسابيع 12-14
- فواتير مشتريات + VAT + أوامر شراء (PO).
- مرتجعات مشتريات.
- حسابات الموردين (A/P Aging).
- الدفعات والأذونات.
- **تقارير:** Purchase Invoices, Vendor Statement, PO Status, Price Comparison, A/P Aging, VAT Report.

### المرحلة 6: التصنيع (Manufacturing) — أسابيع 15-17
- فواتير المواد (BOM).
- أوامر التشغيل (Work Orders).
- تكاليف الإنتاج (Standard vs Actual).
- **تقارير:** Production Cost, Work Order Status, Material Consumption, Variance Analysis, Line Efficiency.

### المرحلة 7: الموظفين (HR) — أسابيع 18-19
- سجلات موظفين، أقسام، درجات وظيفية.
- حضور وانصراف (Attendance).
- مسير رواتب + استقطاعات + بدلات.
- نهاية الخدمة (End of Service).
- **تقارير:** Payroll Sheet, Attendance Report, Leave Balance, End of Service Calculation, Salary Analysis.

### المرحلة 8: CRM — أسبوع 20
- الفرص (Opportunities) وقمع المبيعات (Sales Funnel).
- المهام، المكالمات، الاجتماعات.
- سجل النشاطات.
- **تقارير:** Sales Funnel, Lead Conversion, Rep Performance, Activity Log.

### المرحلة 9: التقارير المتقدمة + Dashboard — أسبوع 21
- **لوحة التحكم الرئيسية (Main Dashboard):** KPIs من كل الوحدات.
- **التنبيهات الذكية:** Alerts قابلة للتخصيص.
- **التوقعات:** Forecasting للمبيعات والأرباح.
- **التقارير المخصصة:** User-defined reports (مستقبلي).

### المرحلة 10: الاختبار والإطلاق — أسبوع 22
- اختبار شامل (Unit + Integration + E2E).
- تحسين الأداء (Optimization).
